# design-sync notes — Fin Finance App UI

This is an **application repo** (Vite + React 19), not a packaged design system. The
sync covers a scoped subset of reusable, prop-driven presentational components from
`frontend/src` plus the `index.css` token layer. Project: `Fin Finance App UI`
(`f68e0507-30c1-470b-9722-2326bdfe7af8`).

## Build setup (synth-entry, no dist/.d.ts)
- No published component dist and no TypeScript — components are JSX default exports.
- A hand-written entry `frontend/.ds-entry.jsx` (gitignored) does named re-exports
  (`export { default as X }`) of each scoped component → `window.FinUI.*`. `export *`
  alone would DROP default exports, so the explicit entry is required.
- Entry lives **under `frontend/`** on purpose: PKG_DIR is resolved by walking up from
  `--entry` to the first `package.json`. The repo-root `package.json` has no `name`,
  which made the `.d.ts` types-root walk overshoot to `/package.json` and crash. Keeping
  the entry under `frontend/` (named "frontend") fixes it. `componentSrcMap` paths and
  `srcDir`/`cssEntry` are therefore relative to `frontend/`.
- Build: `node .ds-sync/package-build.mjs --config .design-sync/config.json
  --node-modules ./frontend/node_modules --entry frontend/.ds-entry.jsx --out ./ds-bundle`
- Emitted `.d.ts` are empty (`[key:string]:unknown`) — synth mode has no real types.
  Consider `cfg.dtsPropsFor` for the core components to give the design agent a real
  prop contract (TODO — see Re-sync risks).

## Environment
- Ubuntu 24.04: chromium needs the `t64` lib variants (`libasound2t64`, `libcups2t64`,
  `libatk1.0-0t64`, `libatk-bridge2.0-0t64`, `libatspi2.0-0t64`, + libgbm1 libcairo2
  libpango-1.0-0 libpangocairo-1.0-0 libxcomposite1 libxdamage1 libxfixes3 libxrandr2
  libnss3 libnspr4 libxkbcommon0 libdrm2 libxshmfence1). Plain `libasound2` is virtual
  and aborts the whole apt install.
- Playwright **1.60.0** pins the cached chromium build (1223).

## Fonts & tokens
- Brand font **Outfit** (the app's base + heading family) is loaded by the app via a
  Google Fonts `<link>` at runtime. The bundle ships it self-contained: woff2 + a local
  `@font-face` CSS at `.design-sync/fonts/outfit.css`, wired via
  `cfg.extraFonts: "../.design-sync/fonts/outfit.css"`. These font files are durable
  (committed), so re-sync needs no network.
- `Roboto Mono` / `Cascadia Code` are only fallback names in `--font-mono` (never loaded
  by the app) → suppressed via `cfg.runtimeFontPrefixes`.
- **[TOKENS_MISSING] is partly a latent app bug:** `KpiCards.css` references
  `--text`, `--text-muted`, `--font-heading` which are NOT defined anywhere in the app
  (index.css uses `--ink`/`--muted`). KpiCard text colors fall back. The `--token`
  warning is a FALSE POSITIVE scraped from a comment in index.css. Fix `--text`/etc.
  inside the KpiCard/KpiCardRow previews with a wrapper that defines them.

## Authoring patterns (calibrated on solo set)
- Previews import from `'frontend'` (the harness shims to `window.FinUI`). Default-export
  components are imported as named (`import { ConfirmModal } from 'frontend'`).
- **Overlay components** (ConfirmModal — `position:fixed; inset:0` overlay) don't center
  in the card harness (modal renders at negative y). Wrap in a `<Frame>` that injects a
  `<style>` neutralizing the overlay (`position:static; background:transparent;
  display:block`) so it renders in normal flow. HelpPanel is also an overlay panel →
  same treatment expected.
- `autoFocus` reliably opens dropdown components in headless (AccountPicker OpenDropdown).
- Use realistic finance content (accounts, reconciliation, budgets), never foo/bar.

## Known render warns (triaged, expected)
- `[TOKENS_MISSING]` for `--text --text-muted --font-heading --token` — see above; the
  first three are KpiCards-only (fixed in preview), `--token` is a comment false positive.

## Re-sync risks
- `.d.ts` are empty stubs — if the design agent needs real prop contracts, add
  `cfg.dtsPropsFor` for the core components (not yet done).
- Outfit is pinned to Google Fonts `v15` hashes in `.design-sync/fonts/`; if the app's
  Google Fonts URL changes weights, refresh the woff2.
- Component set is hand-pinned in `cfg.componentSrcMap`; adding/removing an app component
  does NOT auto-update the sync — edit the map.
- `frontend/.ds-entry.jsx` is gitignored and regenerated by hand; if lost, recreate the
  named re-exports for every component in `componentSrcMap`.
